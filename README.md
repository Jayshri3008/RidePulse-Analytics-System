# RidePulse-Analytics-System
End-to-end data analytics project with machine learning and Power BI dashboard

## Project Overview

RidePulse Analytics System is an end-to-end data analytics project focused on analyzing ride-sharing data to uncover pricing patterns, demand trends, and the impact of surge pricing. The project combines data preprocessing, exploratory data analysis, machine learning, and dashboard visualization.

## Objectives

* Analyze ride demand and pricing behavior
* Understand the impact of surge multiplier on fares
* Identify trends based on time, location, and ride type
* Build a predictive model for ride price estimation
* Create a professional Power BI dashboard for business insights

## Dataset

The dataset used in this project is publicly available on Kaggle (Ride Sharing Dataset). Due to file size limitations, it is not included in this repository.

## Project Workflow

1. Data Collection
2. Data Cleaning and Preprocessing
3. Exploratory Data Analysis (EDA)
4. Feature Engineering
5. Model Building and Evaluation
6. Dashboard Development

## Machine Learning Models

* Linear Regression
* Decision Tree Regressor
* Random Forest Regressor

## Key Insights

* Ride demand varies significantly across different hours of the day
* Surge pricing has a direct impact on fare increase
* Premium ride types have higher average fares
* Weather and time influence ride pricing patterns

## Dashboard

The Power BI dashboard provides:

* Ride demand by hour
* Fare analysis by ride type
* Platform distribution (Uber vs Lyft)
* Surge impact on pricing

## Technologies Used

* Python
* Pandas
* NumPy
* Scikit-learn
* Power BI

## Conclusion

This project demonstrates the complete data analytics lifecycle, from raw data processing to actionable business insights using visualization and machine learning.
